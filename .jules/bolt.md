## 2023-10-25 - [Performance Boost] Optimize O(N^2) connection retain loop with FxHashSet
**Erkenntnis:** Using `Vec::contains` inside a `.retain` loop for connection processing results in an O(N*M) algorithmic bottleneck, particularly when validating large numbers of part connections.
**Aktion:** Replaced the `Vec` collection with a `rustc_hash::FxHashSet` constructed prior to the loop. This changes the lookup cost from O(N) to O(1), improving the overall loop execution to O(M) and yielding a ~280x performance boost in microbenchmarks. Applied the turbofish syntax `collect::<rustc_hash::FxHashSet<_>>()` to ensure types resolve correctly regardless of diff contexts.
## 2026-04-16 - [Performance Boost] Replace O(N) linear search in MediaBrowser with HashMap lookup
**Erkenntnis:** The `MediaBrowser` was performing O(N) linear searches through the entire file list whenever a thumbnail or metadata result was received from background workers. In directories with many files, this created a significant UI thread bottleneck.
**Aktion:** Introduced a `path_to_index: HashMap<PathBuf, usize>` to track the position of each entry in the list. This reduced the lookup complexity from O(N) to O(1). The map is automatically rebuilt during sorting or list refreshes. This optimization scales linearly with the number of background results instead of quadratically with the total number of files.
